Background

왜 이 테스트가 필요한가

현장의 Pain-point 및 해결 방향
  • As-Is 1: 연결조정 금액을 자회사로 배부할 때, 비율(Driver) 산정과 전표 생성을 수작업(Excel)이나 외부 시스템에 의존하는 사례가 빈번함
  • As-Is 2: ERP(FI)로 데이터를 역전송하거나 별도의 CBO 프로그램을 개발하게 되면, 결산 아키텍처와 운영 절차가 매우 복잡해짐
  • To-Be: Group Reporting의 표준 배부 엔진(Automatic Postings)을 활용하면 연결 시스템 내부에서 단순 로직으로 자동 조정 가능
  • 기대 효과: 결산 마감 업무의 표준화 및 자동화를 달성하고 휴먼 에러를 원천 차단함
핵심 아이디어
1. Driver 산출
자회사 간 매출액(Revenue) 비중 계산
⬇️
2. 배부율 생성
연결 단위별 배부율(%) 시스템 내 자동 저장
⬇️
3. 원가 배부
본사 연결 매출원가(COGS) 상계 및 자회사 기표

즉, '매출(Revenue)'을 Driver로 사용하여 동적인 배부율을 계산하고, 수작업 개입 없이 '연결 매출원가'를 배부하는 표준 로직을 검증합니다.

Rule 1
Driver Calculation

Rule 1: Revenue Share(%) 계산

Source Data (Selection: Z_SOG)
  • FS Item: 411100 (Sale of Goods)
  • Cons. unit(연결 대상 법인): [Incl.] ZC1~ZC4, [Excl.] ZHQ
  • Variable: Z_VAR_SOG2 = Total Revenue Amount (전체 매출 합계)
  • Variable matching fields에 Subitem Cat. 포함
↓ 배부율 계산 로직
Revenue Share = SOURCE_GC / Z_VAR_SOG2

(개별 자회사 매출액 ÷ 전체 자회사 총 매출액)

Target Data
  • Target Doc. type: Z3 (배부(30/AUT))
  • 결과 저장 항목: ZS_PC_SHR (통계 FS Item)
  • Breakdown category: QP10
    • 기본 단위 (Base Unit): 분석 필수 (입력 필수, 기본값 허용)
    • ★ 파트너 단위 (Partner Unit): 옵션 분류 (초기화된 값 허용)
  • Posting with Quantities 활성화
  • ★ Partner Unit: ZHQ 필드 지정 (Rule 2 본사 배부 연결고리)
  • 예: 0.15로 저장되며, Rule 2 배부 시 ×100 적용

Rule 2
Distribution Posting

Rule 2: COGS 배부 (본사 ➔ 자회사)

Source Data (Selection: Z_COMMISSION_EXP)
  • 대상 필터링: 매출원가(420490) 배부 대상 데이터만 선택
  • Variable: Z_VAR_COGS = Cost of Sales Amount (본사 매출원가)
  • Variable matching fields에 Consolidation Unit 포함
  • 사용 Driver: Rule 1에서 저장한 ZS_PC_SHR (Quantity) 사용
↓ 배부 금액 계산 로직
Distributed Value = SOURCE_QTY × Z_VAR_COGS × 100

(배부율 × 본사 원천 매출원가 × 100)

Target Data & Posting Lines
  • Target Doc. type: Z4 (배부(30/AUT) / 단일 전표)
  • 전기 라인 (4 Lines) 구성:
    • Line 1 (+) & 2 (-): 자회사별 원가(420490) 배부 및 본사 원가 직접 상계
    • Line 3 (-) & 4 (+): 대차 상계를 위한 Offset 항목(420010D) 기표
  • 결과: 각 법인별 대차 잔액이 '0'으로 완벽하게 자동 정산 완료

Design Points

핵심 설계 포인트

1. 동적 배부율 (Dynamic Driver)

  • Quantities as Drivers
    계산된 비율(매출 비중)을 통계항목의 수량(Quantity)으로 저장하여, 후속 배부 규칙에서 재사용 가능
  • Variable 기반 총합 계산
    자회사(연결단위) 추가 및 매출 변동 시에도 변수(Variable)를 통해 분모가 자동 계산되어 배부율 총합 100% 유지

2. 투명성 및 추적성 (Traceability)

  • Document Type 분리
    Z3(배부율), Z4(배부 결과) 사용으로 원천 데이터(Doc: 00)와 자동 계산 로직을 완벽히 분리
  • 감사 추적성 강화
    수작업 엑셀 없이 시스템 내에서 원천, 비율, 배부 결과의 데이터 계보(Data Lineage)를 명확히 추적 가능
규칙 기반 표준 배부 엔진을 사용하여 운영 효율성과 재무 투명성 동시 확보
